# QR-Thrive External Integration Guide

This guide provides a complete technical reference for integrating an external backend (e.g., Vemtap) with the QR-Thrive API and vice versa. It includes authentication, user synchronization, QR management, and automated provisioning.

---

## 1. Authentication

### Calling QR-Thrive
All requests to the QR-Thrive Integration API must include a secure API Key.
- **Header Name:** `X-API-KEY`
- **Value:** `your_secure_api_key_here` (Generated by Admin in QR-Thrive)

### Receiving Calls from QR-Thrive
When QR-Thrive calls your backend (e.g., for user provisioning), it will include your API Key in the headers.
- **Header Name:** `X-API-KEY`
- **Value:** `${VEMTAP_API_KEY}` (Configured in QR-Thrive `.env`)

## 3. Direction A: Your Backend → QR-Thrive

### A1. User Synchronization
Before acting on behalf of a user, ensure they exist in QR-Thrive.
- **Endpoint:** `POST /integration/users`
- **Payload:** `{ email: string, firstName: string, lastName: string }`
- **Response:** `UserResponse` (Use the `id` for subsequent calls).

### A2. QR Code Creation
- **Endpoint:** `POST /integration/users/{userId}/qr-codes`
- **Payload:** `CreateQRCodeDto` (See `schema.prisma` or API docs for full design/frame JSON structures).
- **Example Data:** 
  - URL: `{ "url": "https://example.com" }`
  - WhatsApp: `{ "phone": "+1234567890", "message": "Hi" }`

### A3. Data Retrieval
- **Get Scans:** `GET /integration/users/{userId}/qr-codes/{id}/scans`
- **Get Form Submissions:** `GET /integration/users/{userId}/qr-codes/{id}/responses`

### A4. SSO Login
Generate a single-use, 15-minute token to log your user into the QR-Thrive dashboard.
- **Endpoint:** `POST /integration/users/{userId}/magic-link`
- **Action:** Redirect the user's browser to the returned `url`.

## 4. Direction B: QR-Thrive → Your Backend (Provisioning)

When a user purchases a bundled plan on QR-Thrive, QR-Thrive will automatically notify your backend to upgrade the user.

### B1. User Provisioning
Your backend must implement this endpoint to receive the provisioning request.
- **Method:** `POST`
- **Path:** `/api/v1/users/provision`
- **Security:** Verify the `X-API-KEY` header.
- **Payload:**
```typescript
interface ProvisionUserDto {
  email: string;      // User's email on QR-Thrive
  firstName: string;  // User's first name
  lastName: string;   // User's last name
  planId: string;     // The ID of your plan (Vemtap Plan ID) bundled in QR-Thrive
}
```

### B2. Fetching Plans (Discovery)
QR-Thrive Admins use this to discover which of your plans they can bundle.
- **Method:** `GET`
- **Path:** `/api/v1/plans?onlyActive=true`
- **Response:** An array of your plan objects (must contain `id` and `name`).

## 5. Configuration (Environment Variables)

Ensure these are set in the QR-Thrive backend `.env`:
- `VEMTAP_API_URL`: Your backend base URL.
- `VEMTAP_API_KEY`: Secret key shared between both backends.
- `FRONTEND_URL`: URL of the QR-Thrive React app (for SSO redirects).
- `API_BASE_URL`: URL of the QR-Thrive API (for SSO redirects).
